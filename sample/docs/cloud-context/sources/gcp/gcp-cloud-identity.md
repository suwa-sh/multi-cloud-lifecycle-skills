# Cloud Identity の概要 {#overview-of-cloud-identity data-text="Cloud Identity の概要" tabindex="-1"}

Cloud Identity は、ユーザーとグループを一元管理する IDaaS（サービスとしての ID）ソリューションです。Cloud Identity を構成すると、Google と他の ID プロバイダ（[Active Directory](https://docs.cloud.google.com/architecture/identity/federating-gcp-with-active-directory-introduction?hl=ja)、[Microsoft Entra ID（旧 Azure AD）](https://docs.cloud.google.com/architecture/identity/federating-gcp-with-azure-active-directory?hl=ja)など）の間で ID を連携させることができます。

Cloud Identity では、組織で使用されているアカウントをさらに細かく管理できます。たとえば、組織内の開発者が Gmail アカウントなどの個人用アカウントを使用している場合、そのアカウントは管理できません。Cloud Identity を導入すると、ドメイン内の全ユーザーのアクセスとコンプライアンスを管理できます。

Cloud Identity を導入する際は、ユーザーとグループそれぞれに対して Cloud Identity アカウントを作成します。これにより、Identity and Access Management（IAM）を使用して各 Cloud Identity アカウントのリソースへのアクセスを管理できます。 Google Cloud
